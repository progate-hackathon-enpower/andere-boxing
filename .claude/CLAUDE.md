# andere-boxing

プロジェクトの技術スタック・アーキテクチャ・セットアップ手順・環境変数は [README.md](README.md) を参照すること。

## コミュニケーション

**重要**: 必ず日本語で応答すること。

- 建設的かつ簡潔な会話を心がける
- 推測で回答せず、不明点は質問する
- コード変更の理由を明確に説明する

## コーディング方針

- **既存のコードスタイルを尊重する** — 周囲のコードに合わせる
- **過度な抽象化を避ける** — 現時点で必要な最小限の実装にとどめる
- **セキュリティを意識する** — OWASP Top 10 に該当する脆弱性を作り込まない

## Git 規約

- コミットメッセージは日本語、`feat:` / `fix:` / `update:` / `chore:` などの prefix を付ける
- PR は短いタイトル + Summary セクションで変更内容を説明する

## Issue 管理

- ユーザーと将来のタスクについて話し合う場面では、**issue を立てるかどうかを積極的に提案する**
- issue 作成の作法は [.claude/rules/issue.md](.claude/rules/issue.md) を参照すること
